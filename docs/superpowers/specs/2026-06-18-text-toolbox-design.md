# 文本处理工具箱设计文档

> 工具 ID：`text-toolbox`  
> 路由：`/text/text-toolbox`  
> 分类：文本处理  
> 设计日期：2026-06-18

## 一、目标

把开发者日常零散的文本处理需求收拢到一个页面：粘贴文本后，一键完成大小写转换、全/半角转换、去重、去空行、排序、字数统计、查找替换等高频操作。运算全部在浏览器本地完成，数据不上传，即开即用。

## 二、需求确认

| 需求项 | 决策 |
|--------|------|
| 交互形态 | 方案 A：单一可编辑文本框 + 变换按钮原地改写 + 实时统计 + 查找替换独立区 |
| 功能范围 | ROADMAP 6 类（大小写/全半角、去重、去空行、排序、统计、查找替换）+ 2 项高频清理（去每行首尾空白、合并连续空白） |
| 大小写 | 大写、小写、首字母大写（Title Case） |
| 全/半角 | 全角→半角、半角→全角（ASCII 字母、数字、标点） |
| 去重 | 按行去重，保留首次出现 |
| 排序 | 按行升序（A→Z）、按行降序（Z→A），码点序、区分大小写（大写排前） |
| 字数口径 | 去除所有空白后的字符数 |
| 撤销/重做 | 多步历史栈，上限 50 步 |
| 宽度 | 标准 `max-w-[720px]`（vertical 单列） |
| 水合 | `client:idle` |

### 4.1 变换按钮组

- 按功能分四行排列：大小写、全/半角、清理、排序。
- 任意变换按钮点击后，将结果**原地写回**文本框，并入撤销栈。
- 多个变换可连续叠加（如 大写 → 去重 → 行升序）。
- 操作控制（撤销/重做/清空/复制）与变换按钮同区。

### 4.2 文本框

- 页面加载后自动聚焦。
- 预填示例文本，打开即可体验。
- 内容变化时实时重算统计。

### 4.3 实时统计（只读）

| 指标 | 口径 |
|------|------|
| 字数 | 去除所有空白字符后的字符数 |
| 字符 | 全部字符数（含空白与换行） |
| 字节 | UTF-8 编码字节数（`TextEncoder`） |
| 行 | 换行符分隔的行数 |

### 4.4 查找替换

- 提供「查找词」「替换词」两个输入框。
- 选项：区分大小写、使用正则。
- 点「替换全部」原地改写文本框内容，并入撤销栈。
- 启用正则时用 `new RegExp(pattern, flags)` 编译；语法错误在按钮下方 `text-error` 内联提示，不执行替换。

### 4.5 操作控制

- **撤销 / 重做**：基于历史栈回退/前进；无可撤销/重做时按钮禁用。
- **清空**：清空文本框并重置历史栈。
- **复制**：复制当前文本框内容，成功走 Toast 反馈。

## 五、核心算法（src/utils/text/text-toolbox.ts）

所有变换为纯函数 `(input: string) => string`，便于独立单测。

### 5.1 大小写

- `toUpperCase(input)` / `toLowerCase(input)`：直接 `String.prototype` 方法（中文等无大小写字符原样保留）。
- `toTitleCase(input)`：按行处理，每行首字符 `toUpperCase`、其余 `toLowerCase`。

### 5.2 全/半角

- `toHalfWidth(input)`：将全角 ASCII（U+FF01–FF5E）映射为半角，全角空格 U+3000 → 普通空格 U+0020。
- `toFullWidth(input)`：反向映射（U+0020–007E → U+FF01–FF5E，普通空格 → 全角空格）。
- 仅作用于 ASCII 范围字符，中文不受影响。

### 5.3 清理

- `removeBlankLines(input)`：删除纯空白行（仅含空格/制表符的行视为空白行），保留有内容的行。
- `dedupeLines(input)`：按行去重，保留首次出现顺序（保序 `Set`）。
- `trimLines(input)`：去除每一行的首尾空白。
- `collapseWhitespace(input)`：每行内部将连续空白（空格/制表符）合并为单个空格，不跨行合并、不删除换行。

### 5.4 排序

- `sortLines(input, 'asc' | 'desc')`：按行排序，使用默认码点序（大写 ASCII 排在小写前），区分大小写；空行不参与排序，统一移到结果末尾。

### 5.5 统计

- `computeStats(input): { chars: number; charsNoSpace: number; bytes: number; lines: number }`
  - `chars`：`input.length`。
  - `charsNoSpace`：去除所有 `\s` 后的长度。
  - `bytes`：`new TextEncoder().encode(input).length`。
  - `lines`：空串返回 0，否则按 `\n` 计数。

### 5.6 查找替换

- `replaceAll(input, find, replace, options: { caseSensitive: boolean; regex: boolean }): { result: string; error?: string }`
  - 普通模式：按字面量全局替换，替换串原样输出（不对 `$` 做特殊解释）。
  - 正则模式：`new RegExp(find, flags)`，`flags` 含 `g`，不区分大小写时追加 `i`；替换遵循 JS 标准 `replace` 语义（支持 `$1`/`$&` 等反向引用）；编译失败时返回 `{ result: input, error: '...' }`，不改动原文。

### 5.7 撤销栈

- `createHistory(limit = 50)`：封装 `{ push(s), undo(): string | null, redo(): string | null, canUndo, canRedo, reset() }`。
- `push`：在中段（已 undo 过）插入新状态时，先截断后续分支再 push（标准 undo 语义）；超过 `limit` 时丢弃最早记录。
- 初始化：组件挂载时以初始示例文本 `push` 一次作为历史起点，此时 `canUndo` 为 `false`（初始状态不可撤销）；用户每次变换前 `push` 当前文本。

## 六、错误处理

| 场景 | 处理方式 |
|------|----------|
| 查找替换正则语法错误 | 替换按钮下方 `text-error` 内联提示「正则表达式语法错误：{原因}」，不执行替换 |
| 空输入 | 统计显示全 0，不报错；变换按钮点击对空内容无效果 |
| 复制失败 | Toast「复制失败，请重试」 |
| 超大文本（>100 万字符） | 仍可处理（纯字符串操作）；统计与变换同步执行，无需 Worker |

## 九、性能与安全

- 全部纯字符串操作，无第三方库。
- 禁用 `eval`/`Function`；正则用 `new RegExp` 包 `try-catch`。
- 变换为同步操作，单次处理百万级字符仍在毫秒级，无需 Web Worker。
- 统计用 Vue `computed` 依赖文本框内容自动 memo，无需手动防抖。
