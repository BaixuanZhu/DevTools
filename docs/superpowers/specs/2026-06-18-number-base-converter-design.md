# 进制转换器设计文档

> 工具 ID：`number-base-converter`  
> 路由：`/text/number-base-converter`  
> 分类：文本处理  
> 设计日期：2026-06-18

## 一、目标

实现一个浏览器端即开即用的进制转换工具，支持二进制、八进制、十进制、十六进制之间的批量互转，并提供逐位二进制预览。运算全部在本地完成，数据不上传。

## 二、需求确认

| 需求项 | 决策 |
|--------|------|
| 支持进制 | 2 / 8 / 10 / 16（ROADMAP 基线） |
| 数字范围 | 任意大整数，使用 `BigInt` |
| 负数 | 支持，二进制按补码展示 |
| 位宽规则 | 自动：正数取最小位宽；负数取能容纳补码的最小 8 位倍数 |
| 交互模式 | 批量多行输入 + 实时联动转换；点击行展开二进制位图预览 |
| 二进制位序 | 大端（bit N … bit 0） |

### 4.1 输入区

- 使用 `SelectListbox` 选择输入进制，默认「十六进制」。
- 大文本框逐行输入，每行一个数字。
- 输入即转换，无需点击按钮。
- 空行直接忽略，不报错。

### 4.2 结果表格

- 表头：行号、二进制、八进制、十进制、十六进制。
- 每行对应输入的一行有效数字。
- 点击行在表格下方展开/收起二进制位图预览。
- 仅最后选中的行展开预览。

### 4.3 二进制位图预览

- 按大端展示：`bit N` 在左，`bit 0` 在右。
- 每 4 位（一个 nibble）用空格分组；每 8 位再用中括号 `[...]` 包裹，便于对照十六进制字节。
- 表格「二进制」列显示最小位宽（去掉前导零，至少 1 位）。
- 预览面板为了对齐 nibble/字节：正数左侧补零至 4 的倍数；负数使用最小 8 位倍数位宽的补码。
- 示例：`0x1A3F` 表格列显示 `1101000111111`，预览面板显示 `[0001][1010][0011][1111]`；`-1` 预览显示 `[11111111]`。

### 4.4 操作栏

- **清空**：清空输入和结果。
- **复制全部结果**：复制所有行的转换结果（按行输出 `原始值 = 二进制 / 八进制 / 十进制 / 十六进制` 格式）。

## 五、核心算法

### 5.1 解析（字符串 → BigInt）

```ts
function parseNumber(input: string, base: number): bigint | null
```

- 去除前后空白。
- 支持可选前导符号 `+` / `-`。
- 校验字符是否全部属于当前进制允许字符。
- 使用 `BigInt(parseInt(input, base))` 不可行（会丢失大整数精度），应逐位用 `BigInt` 累加。

### 5.2 转换（BigInt → 各进制字符串）

```ts
function convertNumber(value: bigint, base: number): string
```

- 使用 `value.toString(base)` 生成对应进制小写字符串。
- 八进制、二进制不添加前缀；十六进制不添加 `0x` 前缀（保持结果纯净，便于复制）。

### 5.3 补码位宽计算

```ts
function getComplementBitWidth(value: bigint): number
```

- 正数：`max(1, ceil(log2(value + 1)))`。
- 负数：找到最小 `n`（8 的倍数），使得 `value` 能被 `n` 位补码表示，即 `-2^(n-1) ≤ value < 0`。
- 例如 `-1` → 8 位；`-128` → 8 位；`-129` → 16 位。

### 5.4 二进制位图字符串

```ts
function formatBinaryPreview(value: bigint, bitWidth: number): string
```

- 先按位宽生成补码二进制字符串（负数用 `2^n + value`）。
- 每 4 位插入一个空格；每 8 位整体用 `[...]` 包裹。

## 六、错误处理

| 场景 | 处理方式 |
|------|----------|
| 输入包含当前进制不允许的字符 | 输入框下方显示：「第 X 行包含无效字符，请检查当前进制」 |
| 空输入 | 结果区显示占位提示，不报错 |
| 单行超长（如 > 10 000 字符） | 跳过该行并提示「第 X 行过长，已跳过」 |
| 复制失败 | 通过 `useCopy` 触发 Toast「复制失败，请重试」 |

## 九、性能考虑

- 纯 `BigInt` 运算，无第三方库。
- 输入文本按行处理，单行超长时截断，避免极端输入拖慢页面。
- 转换使用 `requestAnimationFrame` 或简单的防抖（150 ms）避免连续输入时频繁重算。
